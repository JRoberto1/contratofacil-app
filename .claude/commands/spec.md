# /spec — Spec-Driven Development

Escreva a spec ANTES de qualquer código.

## Instrução
Crie um documento de spec com: visão geral, problema, escopo (inclui/não inclui), critérios de aceitação, impacto técnico.
Salve em docs/specs/[nome].md. Aguarde aprovação antes de implementar.

## Anti-Rationalization
❌ "É urgente" → Urgência é motivo para ter spec, não para pular
❌ "É simples" → Spec descobre complexidade antes do código
